# 오프린트미 GNB 개편 프로토타입

오프린트미(ohprint.me) **상단 글로벌 네비게이션(GNB) 개편 기획**을 공유하기 위한 인터랙티브 프로토타입입니다.
대카테고리 추가·삭제·이름변경, 중카테고리(그룹)·소카테고리(항목) 편집, 그리고 **변경 전 / 변경 후 비교 뷰**를 제공합니다.

> 기능 동작용이 아닌 **기획 커뮤니케이션용** 프로토타입입니다. 링크 클릭 시 실제 페이지로 이동하지 않습니다.

---

## ✨ 주요 기능

- **4가지 뷰 모드**
  - 변경 전/후 비교 (2분할 나란히 보기)
  - 편집 모드 (편집 UI + 실시간 미리보기)
  - 개편안만 보기 / 현재만 보기
- **대카테고리 편집**: 추가 / 삭제 / 이름변경 / 순서변경 / 뱃지(HOT·NEW·SALE) 설정
- **중카테고리(그룹) 편집**: 그룹명 변경 / 추가 / 삭제
- **소카테고리(항목) 편집**: 추가 / 삭제 / 이름변경 / 뱃지 설정
- **변경사항 요약**: 추가·삭제·이름변경 카테고리, 항목 변경 개수 자동 계산
- **항목 일괄 추가**: 엑셀·노션에서 복사한 항목 목록을 한 줄에 하나씩 붙여넣어 한 번에 추가
- **JSON 내보내기 / 불러오기**: 작업 내용 저장 · 공유 · 불러오기
- **풀폭 드롭다운**: 실제 ohprint.me처럼 전체 화면 폭으로 펼쳐지는 드롭다운
- **신규 대카테고리 하이라이트**: 개편안에서 추가된 항목은 NEW 뱃지 + 파란색으로 표시
- **픽셀 단위 디자인 재현**: 실제 오프린트미 컬러(오렌지 #FFA02E + 블루 #1F7DFF), 뱃지 스타일, 간격, 볼드 폰트까지 동일하게 구현

## 🚀 시작하기

### 사전 요구사항
- Node.js 18 이상
- npm 또는 yarn

### 설치 및 실행

```bash
# 저장소 클론
git clone <your-repo-url>
cd ohprint-gnb-prototype

# 의존성 설치
npm install

# 개발 서버 실행
npm run dev
```

브라우저에서 `http://localhost:5173`으로 접속하세요.

### 프로덕션 빌드

```bash
npm run build
npm run preview
```

빌드 결과물은 `dist/` 폴더에 생성됩니다. 이 폴더를 GitHub Pages, Vercel, Netlify 등에 배포할 수 있습니다.

## 🖱️ 사용법

### 1. 변경 전/후 비교 모드
기본 뷰입니다. 좌우로 현재 GNB와 개편안 GNB를 나란히 볼 수 있습니다. 각 GNB의 대카테고리에 마우스를 올리면 드롭다운이 펼쳐집니다.

### 2. 편집 모드
왼쪽에서 개편안 구조를 편집하면 오른쪽 미리보기에 즉시 반영됩니다.
- 각 대카테고리 우측 **[펼치기]** 버튼으로 중카테고리(그룹)를 확인·편집할 수 있습니다.
- 각 그룹의 **[항목 N개]** 버튼으로 소카테고리 항목을 펼쳐 편집할 수 있습니다.
- 왼쪽 **▲▼** 버튼으로 대카테고리 순서를 변경할 수 있습니다.

### 3. JSON 내보내기 / 불러오기
작업 내용을 JSON 파일로 저장하면 다른 기기/브라우저에서 불러올 수 있습니다. 여러 개편안을 각각 파일로 관리할 수 있어요.

## 🔧 다른 개편 기획에 재사용하기

다른 프로젝트(예: snaps.com, 다른 카테고리 개편)에도 재사용하려면 `OhprintGnbPrototype.jsx` 파일 상단의 **`INITIAL_DATA`** 상수를 교체하면 됩니다.

```js
const INITIAL_DATA = {
  categories: [
    {
      id: "your-category-id",
      name: "카테고리명",
      badge: "HOT", // 선택: HOT, NEW, SALE
      groups: [
        {
          id: "your-group-id",
          name: "그룹명", // 빈 문자열이면 그룹명 숨김
          items: [
            { id: "item-1", name: "항목명", badge: "HOT" }
          ]
        }
      ]
    }
  ]
};
```

초기 개편안이 다르다면 `getInitialAfterData()` 함수도 함께 수정해주세요.
